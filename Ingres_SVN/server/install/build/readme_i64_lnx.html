1.0 Welcome

This readme contains information specific to Ingres® running under Itanium Linux.

2.0 Operating System Support

The following operating system is supported (as of the publishing of this document):

Any Linux distribution based on Kernel 2.4.21+ or 2.6.x and glibc 2.2.5+ or 2.3.x C-language libraries

Ingres supports these operating systems for the duration of their life cycle (as determined by the operating system's manufacturer or until Ingres Corporation announces that it is dropping support). For the latest information about supported operating systems, visit http://www.ingres.com/support.

2.1 Preprocessor and Compiler Support

GCC v3.2 and upwards (GNU Compiler Collection, C)

4.0 Installation Considerations

4.1 RPM Support

As of the time this readme was published, the installation of Ingres requires RPM with a version in the range 3.0.6 through 4.2.2, except for RPM version 4.1. If RPM 4.1 is installed, it must be upgraded to version 4.1.1 or higher before you can install Ingres.

RPM 4.1 fails to set environment variable RPM_INSTALL_PREFIX. Attempting to install Ingres with RPM 4.1 will lead to numerous "cannot find file" errors, and a failed install. For further details of this RPM bug, see http://bugzilla.redhat.com/bugzilla/show_bug.cgi?id=75057. 

On RedHat Linux the rpmbuild package is required to rename ingres packages using the iirpmrename utility.

Note: When installing multiple Ingres RPM packages, it is important that you install the "ingres2006" (core) package before any other package.

Upgrade Installs Using RPM

These instructions allow you to upgrade ANY Ingres r3 release prior to 3.0.2(any.lnx/116) using RPM and avoid BUG 113652/Problem INGINS300 and BUG 114752. This document is to be used in addition to the Ingres Migration Guide.

Note: For the purposes of this document build r3 3.0.1(int.lnx/103) has been used as the version of Ingres being upgraded but the principle is the same for upgrading all versions prior to r3 3.0.2(any.lnx/116)

Upgrading Ingres r3 3.0.1(int.lnx/103) RPMs

  1. Make sure the Ingres instance you wish to upgrade is completely shutdown and that you have taken a backup as described in the Ingres Migration Guide.

  2. Obtain the list of Ingres r3 3.0.1(int.lnx/103) RPM for the instance which is to be upgraded. 
    rpm -qa | grep ca-ingres | grep 103

    All of these packages must be upgraded at once. Any attempt to perform an upgrade on any single or smaller group of packages will fail.

  3. Login as root and CD to the directory containing the RPM to be used to perform the upgrade. Make sure variable II_CONFIG is not set.

  4. Run rpm -ivh to install the new RPMs over the existing installation. If the installation was originally installed using --prefix to change II_SYSTEM from the default, then it MUST also be used for the upgrade.
    For default installation location: 
    rpm -ivh --replacefile --replacepkgs \
    
     <complete list of RPM files to be install>

    Any other installation location: 

    rpm -ivh --replacefiles --replacepkgs --prefix <II_SYSTEM> \
    
     <complete list of RPM files to be install>

  5. Remove the RPM package information for the old installation (103) Check we can find the correct packages: 
    rpm -qa | grep ca-ingres | grep 103

    If the list is OK, then we can remove them: 

    rpm -qa | grep ca-ingres | grep 103 | xargs rpm -e justdb

Note: If you tried using rpm Uvh instead of the instructions above, binary and other installed files (for example, iimerge) may be removed when rpm -Uvh is used with --prefix to upgrade and installation. Should this occur simply re-run the same command again using additional flags --replacefiles & --replacepkgs and all the missing files will be replaced. Any database or configuration files will not be affected.

4.2 Documentation

Ingres documentation can be installed on Linux systems by installing the documentation rpm file supplied with the distribution.

4.3 Mounting the CD-ROM

Most Linux distributions automatically mount the CD-ROM as soon as it is placed in the drive. Running the command "mount" returns an entry for /dev/cdrom if it has been mounted.

If it is not mounted, then it can be done manually. In general, the CD-ROM should either be mounted to /mnt/cdrom or /media/cdrom, depending on the distribution. The following instructions are an example of how to mount the CD-ROM on a RedHat Itanium Linux system:

As root execute this command: 

mount /dev/cdrom /mnt/cdrom

For further information, see the Getting Started guide.

4.4 Installing on a System without a C Compiler

This version of Ingres requires that the DBMS server executables be re-linked so that utilities such as CKPDB and ROLLFORWARDDB will function for users other than "ingres". If a C compiler is present, the server is re-linked at install time. If not, a warning is issued and the install continues as usual. If users other than "ingres" require the use of these utilities, the DBMS server must be re-linked by hand with the following command: 

iilink noudts

A C compiler must be installed for this command to complete successfully.

4.5 Shared Library Path

The library variable for Itanium Linux is LD_LIBRARY_PATH.

4.6 Visual Tools

The Visual DBA suite is not included as part of the Ingres 2006 Open Source for Itanium Linux. However, these tools can be deployed on a Windows PC against Itanium Linux.

4.7 Web Server Support

Ingres for Itanium Linux supports the following web server:

Apache HTTP server 1.3

4.8 Protocols Supported

TCP/IP Native

4.9 Raw Transaction Log Files and Raw Database Locations Not Supported on Linux

Research has shown that raw transaction log files and raw database locations on Linux do not provide performance benefits. To simplify administration, a decision has been made to not support this feature on Linux.
